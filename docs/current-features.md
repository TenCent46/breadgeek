# BreadGeek 現在の機能一覧（AI共有用ドキュメント）

## プロジェクト概要

**BreadGeek**はクリエイター向けサービス販売マーケットプレイスとして開発が始まったが、現在「パン教室・料理教室向け」に特化する方向で差別化を検討中。

### 技術スタック
- Next.js 16 (App Router) + React 19 + TypeScript 5
- Tailwind CSS 4（CSS custom propertiesでデザイントークン管理）
- Lucide React（アイコン）
- Noto Sans JP（フォント）
- データベース未接続（React Context + モックデータで全機能が動作）

### 現在の課題
- 機能セットが汎用サービスマーケットプレイス「MOSH」に酷似しており、差別化が不十分
- パン教室・料理教室に特化した機能がなく、競合との明確な違いがない

## 現在のページ構成と機能詳細

### 1. ダッシュボードホーム (`/dashboard`)

**概要**: ログイン後の最初の画面。ビジネス全体の概況を一目で把握できる。

**表示内容**:
- 統計カード4枚: 今月の売上（前月比%表示）、今月の予約数、累計顧客数、平均評価
- はじめましょうチェックリスト: プロフィール設定、サービス作成、決済設定、ブログ記事投稿（進捗バー付き、完了後は非表示）
- 直近の予約リスト: 今日以降の予約を日時順に5件表示（顧客名、サービス名、日時、ステータスバッジ）
- 7日間の売上ミニバーチャート: CSS/SVGで描画、日別の売上を視覚化
- クイックアクション: サービス作成、ブログ投稿、メッセージ配信、売上確認への直リンクボタン4つ
- アクティビティフィード: 予約・レビュー・売上の最新アクティビティを時系列で表示

**データソース**: services, bookings, customers, reviews, sales, articles すべてのエンティティから集計

### 2. サービス管理

#### 2.1 サービス一覧 (`/dashboard/services`)

**概要**: 出品中の全サービスを管理する画面。

**機能**:
- グリッド表示 / リスト表示の切替ボタン
- 検索バー（サービス名でフリーテキスト検索）
- フィルター:
  - タイプ別: クラス・イベント / プライベート / オンライン / サブスクリプション / コンテンツ販売
  - ステータス別: 公開中 / 下書き / アーカイブ
- 各サービスカードに表示: サムネイル（プレースホルダー）、タイプアイコン+バッジ、タイトル、価格、ステータス、スケジュール数、定員
- アクション: 編集（編集ページへ遷移）/ 複製（下書きとしてコピー作成）/ 削除（確認モーダル付き）
- 検索結果件数の表示

#### 2.2 サービス新規作成 (`/dashboard/services/new`)

**概要**: 6ステップウィザードで新しいサービスを作成。

**ステップ構成**:
1. **タイプ選択**: 5種類のサービスタイプから選択（カード形式、アイコン付き説明文）
   - クラス・イベント: 日程固定のグループレッスン・ワークショップ
   - プライベート: 1対1の個別レッスン
   - オンライン: Zoomを使ったオンラインレッスン
   - サブスクリプション: 月額制サービス
   - コンテンツ販売: レシピ動画・PDF等のデジタルコンテンツ
2. **基本情報**: タイトル、説明文（textarea）、カテゴリ選択（6カテゴリ: パン教室/料理教室/お菓子教室/飲料/フードビジネス/その他）
3. **料金設定**: 価格（円）、キャンセルポリシー選択（柔軟/普通/厳格）
4. **スケジュール**: サービスタイプに応じて異なるフィールド
   - クラス・イベント: 日付、開始時間、終了時間、定員、会場
   - プライベート: 所要時間のみ（空き時間は別ページで設定）
   - オンライン: 日付、開始時間、終了時間、Zoom URL
   - サブスクリプション: 請求サイクル（月額/年額）
   - コンテンツ販売: スケジュール不要
5. **設定**: URLスラッグ（タイトルから自動生成）、公開ステータス（下書き/公開）
6. **確認**: 全入力内容のレビュー画面 → 作成ボタン

#### 2.3 サービス編集 (`/dashboard/services/[id]`)

**概要**: 既存サービスの全フィールドを編集。

**機能**:
- 基本情報の編集（タイトル、説明、カテゴリ）
- 料金・キャンセルポリシーの編集
- スケジュールの追加・削除（サービスタイプに応じた入力フィールド）
- 公開ステータスの変更
- 保存ボタン

### 3. 予約管理

#### 3.1 予約一覧 (`/dashboard/bookings`)

**概要**: 顧客からの予約を管理する画面。カレンダーとリストの2表示。

**カレンダービュー**:
- 月間カレンダーグリッド（月送り対応）
- 予約がある日にドットインジケーター表示
- 日付クリックで右パネルにその日の予約リスト表示
- 各予約に顧客名、サービス名、時間、ステータスバッジ

**リストビュー**:
- ステータスフィルターチップ: すべて / 確認済み / 保留中 / キャンセル済み / 完了
- テーブル表示: 日時、サービス名、顧客名、金額、ステータス、アクション

**予約詳細（スライドオーバーパネル）**:
- 顧客情報（名前、メール）
- サービス情報（名前、タイプバッジ）
- 日時
- 金額
- ステータス変更ボタン: 確認する / キャンセルする / 完了にする
- メモ欄（テキストエリア + 保存ボタン）

**統計バー**: 今日の予約数、保留中の予約数、今月の予約数

#### 3.2 プライベート予約設定 (`/dashboard/bookings/private-status`)

**概要**: プライベートレッスンの空き時間と予約ルールを設定する画面。

**週間スケジュールエディタ**:
- 月〜日の各曜日ごとにON/OFFトグル
- 各曜日に時間枠を複数追加可能（開始時間、終了時間）
- 時間枠の追加・削除ボタン

**予約設定**:
- バッファ時間（分単位入力: 予約間の休憩時間）
- 1日の最大予約数
- 自動承認トグル（ONで自動確認、OFFで手動確認）

**直近のプライベート予約表示**: 今後のプライベート予約一覧

### 4. プロフィール管理

#### 4.1 プロフィールリンク (`/dashboard/profile/link`)

**概要**: 公開プロフィールページをブロックベースで編集。Linktree的な機能。

**ブロックタイプ**:
- テキストブロック: フリーテキスト入力
- 画像ブロック: URL入力で画像表示
- リンクブロック: URL + ラベル
- サービスブロック: サービスIDから自動表示
- SNSブロック: SNSリンク
- 区切り線ブロック: 視覚的なセパレーター

**機能**:
- ドラッグ風の並べ替え（上/下ボタン）
- ブロック追加ボタン
- ブロック削除ボタン
- 右側にモバイルプレビュー表示（スマートフォンモックアップ内にリアルタイムプレビュー）
- 公開/下書き切替トグル

#### 4.2 プロフィール編集 (`/dashboard/profile/edit`)

**概要**: クリエイター自身のプロフィール情報を編集。

**編集フィールド**:
- プロフィール画像（カメラアイコンでアップロード）
- 表示名
- 肩書き（職業・専門分野）
- 自己紹介文（500文字制限、文字数カウンター表示）
- 専門タグ（最大10個、入力→追加ボタン、×で削除）
- カテゴリ選択
- 所在地（都道府県プルダウン + 市区町村テキスト入力）
- メールアドレス
- 電話番号
- SNSリンク（プラットフォーム名 + URL、動的に追加・削除可能）

#### 4.3 特典管理 (`/dashboard/profile/benefits`)

**概要**: 顧客ロイヤルティプログラムの特典を管理。

**統計カード**: 特典総数、アクティブな特典数、総利用回数

**特典カード表示**:
- ティアバッジ（ブロンズ/シルバー/ゴールド/プラチナ）
- タイトル
- 説明文
- 必要ポイント
- 利用回数
- アクティブ/非アクティブトグル
- 編集/削除ボタン

**作成・編集モーダル**: タイトル、説明文、ティア選択、必要ポイント入力、アクティブトグル

### 6. 顧客管理（CRM）

#### 6.1 顧客一覧 (`/dashboard/customers`)

**概要**: サービスを購入した顧客のデータベース。

**ティアフィルター**: すべて / コアファン / レギュラー / トライアル / 休眠

**テーブル表示**: チェックボックス、アバター、名前、ティアバッジ、購入回数、累計金額、最終購入日

**顧客詳細（スライドオーバーパネル）**:
- プロフィール（アバター、名前、ティアバッジ、メール）
- 統計カード3枚（購入回数、累計金額、最終購入日）
- タグ管理（追加・削除可能）
- メモ欄（テキストエリア + 保存ボタン）
- 購入履歴テーブル（日付、サービス名、金額）

**その他**: CSV出力ボタン（プレースホルダー）、検索バー

#### 6.2 コンタクトリスト (`/dashboard/customers/contacts`)

**概要**: まだ購入していないリード（見込み客）を管理。

**テーブル表示**: 名前、メール、流入元（Instagram/ウェブサイト/LINE/紹介）、タグ、登録日

**機能**:
- コンタクト追加モーダル（名前、メール、流入元、タグ）
- CSV インポート/エクスポートボタン（プレースホルダー）
- タグフィルター、流入元フィルター
- コンタクト詳細スライドパネル（タグ管理、削除）

#### 6.3 顧客向けメッセージ配信 (`/dashboard/customers/messages`)

**概要**: 既存顧客にメッセージを一斉配信。

**メッセージ一覧**: 件名、チャネルバッジ（Email/LINE）、ステータスバッジ（下書き/予約/送信済み/送信中）、配信先人数、開封率、クリック率、配信日

**作成・編集モーダル**:
- チャネル切替（Email / LINE）
- 件名（Emailのみ）
- 本文テキストエリア（テンプレート変数 `{{name}}` 対応）
- 配信先選択（顧客ティア別: 全員/コアファン/レギュラー/トライアル）
- 配信方法（今すぐ / 予約配信 → 日時指定）
- 下書き保存 / 送信ボタン

#### 6.4 コンタクト向けメッセージ配信 (`/dashboard/customers/contact-messages`)

**概要**: コンタクト（見込み客）向けのメッセージ配信。Betaバッジ付き。

**ウェルカムメール設定**:
- 有効/無効トグル
- 件名、本文
- 遅延時間（登録後何分後に送信するか）
- 保存ボタン

**メッセージ一覧と作成**: 顧客向けと同様の機能（タグベースのセグメント配信）

### 11. 設定 (`/dashboard/settings`)

**概要**: 7タブ構成の設定画面。

**タブ一覧と各タブの機能**:

1. **アカウント設定**: メールアドレス変更、パスワード変更、言語設定、アカウント削除
2. **ビジネス設定**: ビジネス名、業種選択、説明文、ウェブサイトURL
3. **本人確認**: 確認ステータス表示、身分証明書アップロード（ファイル選択）、送信ボタン
4. **口座情報**: 銀行名、支店名、口座種別（普通/当座）、口座番号、口座名義の入力フォーム
5. **特定商取引法**: 販売業者名、代表者名、所在地、電話番号、メール、商品代金以外の費用、返品/交換ポリシー → プレビュー機能
6. **外部連携**: Stripe/LINE/Zoom/Googleカレンダー の接続カード（接続/解除トグル）
7. **通知設定**: イベント種別 × チャネル（メール/プッシュ/LINE）のトグルグリッド
   - イベント: 新規予約/予約キャンセル/決済完了/新規レビュー/メッセージ受信

## 共通UIパターン

- **カラーパレット**: パン焼き色をイメージした温かみのある配色（チョコレートブラウンの主色、ゴールデンアンバーのアクセント、クリーム色の背景）
- **レイアウト**: 左サイドバー固定 + ヘッダー（パンくずナビ）+ メインコンテンツエリア
- **モーダル**: 中央配置の作成・編集ダイアログ
- **スライドオーバー**: 右端から出現する詳細パネル（顧客詳細、予約詳細）
- **バッジ**: 色分けされたステータス/タイプ/ティアのラベル
- **空状態**: アイコン + メッセージ + アクションボタンの統一パターン
- **統計カード**: アイコン + 数値 + ラベルの統一レイアウト
- **検索・フィルター**: 検索バー + ドロップダウンフィルターの共通コンポーネント

## 現在の問題点（差別化の観点）

### MOSHとの類似性
以下の機能は汎用サービスマーケットプレイス「MOSH」とほぼ同じ構成:
1. サービス5タイプ（クラス/プライベート/オンライン/サブスク/コンテンツ）
2. 予約管理（カレンダー + リスト）
3. 顧客管理（ティア分類）
4. メッセージ配信
5. ブログ機能
6. 売上管理
7. プロフィールリンク（Linktree的機能）
8. ワークフロー自動化

### パン教室特化で差別化できる可能性のある領域（検討用）
- レシピ管理・共有機能
- 材料リスト・買い物リスト自動生成
- レッスンのステップバイステップ管理
- 生徒の作品ギャラリー
- 難易度レベル・アレルギー情報の管理
- 発酵タイマー・温度管理ツール
- 季節のパン・旬の食材連動
- 教室の設備・オーブン情報管理
- 生徒のスキルレベル追跡
- レッスン後のフォローアップ（焼き上がり写真共有）
